## What this documentation demonstrates

This site is a portfolio piece showcasing senior-level technical writing for a developer-facing SDK. Every page is written to Microsoft Style Guide (MSTP) standards:

- **Active voice throughout.** "The SDK requests a token" — not "a token is requested."
- **Second person.** "You configure the client once" — not "the client should be configured."
- **Task orientation.** Each guide answers a specific question and gets out of the way.
- **No minimizers.** No "simply," "just," "easy," or related language that sets expectations for the reader.
- **Accessible code.** Every code block has a language tag, inline comments explaining intent (not syntax), and a copy button.
- **Full coverage of failure states.** Authentication, error handling, and retry pages are as detailed as the happy path.

## Technical features

- **Dark and light mode** — respects `prefers-color-scheme`; toggle persists in `localStorage`
- **Syntax-highlighted code** — CSS token classes; no runtime JS parsing
- **Tab-based language switcher** — Python / Node.js / cURL on every sample
- **Responsive layout** — collapsible sidebar on mobile
- **Copy-to-clipboard** on every code block
- **Zero external dependencies** — no CDN calls, no JavaScript frameworks
